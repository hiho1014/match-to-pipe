# 2026-01-20 우선순위 결정: 스테이지 전환 vs 배선 오류 수정

**작성일**: 2026-01-20  
**작성자**: Antigravity  

---

## 1. 🧐 고민 상황

현재 두 가지 작업이 남아있습니다.
1. **스테이지 전환 구현**: 퍼즐을 다 풀었을 때 다음 단계로 넘어가는 기능 (현재 없음)
2. **배선 오류 수정**: 연결에 영향을 주지 않는(쓸데없는) 배선까지 빛나는 현상 수정

**"무엇을 먼저 할 것인가?"** 에 대한 분석입니다.

---

## 2. 📊 선택지 분석

### 옵션 A: 스테이지 전환 먼저 구현 (추천 ⭐️)

게임을 '게임답게' 만드는 필수 기능입니다.

*   **설명**:
    *   현재는 모든 전구를 켜도 아무 일이 일어나지 않습니다.
    *   모든 전구가 켜졌는지 감지하는 `승리 조건(Win Condition)` 체크 로직을 추가합니다.
    *   성공 시 "성공!" 메시지나 연출 후 **자동으로 다음 스테이지**로 넘어가게 합니다.
*   **장점**:
    *   게임의 **목적성**이 생깁니다. (플레이어가 "깼다!"는 느낌을 받음)
    *   테스트할 때 흐름이 끊기지 않고 1→2→3... 연속 플레이가 가능해집니다.
*   **단점**:
    *   당장 눈에 거슬리는 배선 문제는 그대로 보입니다.

### 옵션 B: 배선 오류 먼저 수정

시각적인 완성도를 높이는 작업입니다.

*   **설명**:
    *   현재 전구까지 가는 **최단 경로(또는 유효 경로)** 외에, 곁가지로 뻗어나간 막힌 길까지 빛나는 현상을 봅니다.
    *   Graph Pruning(가지치기) 알고리즘을 개선하여, **진짜 연결에 기여하는 파이프만** 빛나게 합니다.
*   **장점**:
    *   플레이어가 "이게 왜 켜지지?" 하는 혼란을 줄입니다.
    *   시각적으로 깔끔하고 완성도 있어 보입니다.
*   **단점**:
    *   이걸 고쳐도 여전히 게임을 클리어했을 때 아무런 반응이 없습니다. (수동으로 다음 버튼 눌러야 함)
    *   알고리즘 수정 과정에서 실수하면 멀쩡한 경로가 안 켜지는 버그가 생길 수 있습니다 (리스크 있음).

---

## 3. ⚖️ 비교 및 제안

| 항목 | 스테이지 전환 (Core Loop) | 배선 오류 수정 (Polish) |
| :--- | :--- | :--- |
| **중요도** | 🚨 **매우 높음** (필수) | ⚠️ 중간 (품질 향상) |
| **만족감** | "게임이 진행된다!" | "깔끔해졌다" |
| **리스크** | 낮음 (기능 추가) | 중간 (기존 로직 건드림) |

### 💡 저의 추천: **옵션 A (스테이지 전환)**

**이유**:
1. **게임의 생명은 흐름입니다.** 아무리 예쁜 배선이라도, 클리어했을 때 반응이 없으면 김이 빠집니다.
2. **배선 오류는 치명적이지 않습니다.** "좀 지저분하게 켜지네?" 정도이지, 게임을 못 쪼개는 버그는 아닙니다. 반면 스테이지 전환이 없는 건 게임 미완성입니다.
3. **구조적 순서**: 게임의 뼈대(스테이지 넘어가기)를 완성한 뒤, 디테일(배선 시각화)을 다듬는 것이 개발 순서상 효율적입니다.

## 4. 🚀 향후 계획 (Roadmap)

만약 **옵션 A**를 선택하신다면 다음과 같이 진행하겠습니다:

1. `CheckWinCondition` 함수 추가 (모든 전구 켜졌는지 확인)
2. 승리 시 간단한 연출 (예: 전체 깜빡임 또는 자물쇠 해제 소리 등)
3. 딜레이 후 다음 스테이지 데이터 로드 기능 구현
4. (그 후에) 배선 알고리즘 다듬기
